⌨️ Command Line Cheat Sheet

Keep this next to your keyboard. The commands you'll use all year to build & run Node/Express apps. Print it.

Where am I? (Navigation)

pwdPrint the folder you're in ("print working directory").
lsList files in this folder. (dir on Windows cmd.)
ls -laList everything, including hidden files like .env.
cd folderGo into folder.
cd ..Go up one folder.
cd ~Go to your home folder.

Make / move / delete files

mkdir projectMake a new folder called project.
touch app.jsCreate an empty app.js. (Or: echo "" > app.js.)
cp a.js b.jsCopy a.js to b.js.
mv old.js new.jsRename or move a file.
rm file.txtDelete a file. No recycle bin — be careful!
rm -r folderDelete a whole folder and its contents.

Node & npm (the important ones)

node -vCheck Node is installed (prints the version).
node app.jsRun your server / script.
npm init -yCreate a package.json (accept defaults).
npm installInstall every dependency listed in package.json (creates node_modules).
npm install expressAdd a package (e.g. Express) and save it.
npm install --save-dev nodemonAdd a dev-only tool (used while coding, not in production).
npm startRun the "start" script from package.json.
npx nodemon app.jsRun a dev tool once without installing it globally.

When the server is running

Ctrl + CStop the running server. Use this before re-running.
↑ (up arrow)Recall your last command. Press again for older ones.
TabAuto-complete a file/folder name (saves typing).
clearClear the screen. (cls on Windows cmd.)
Ctrl + C twiceIf one Ctrl-C doesn't stop it, press again.
💡 The #1 beginner mistake: forgetting to stop the old server. If you see EADDRINUSE ("port already in use"), it means a previous node app.js is still running. Go to that terminal and press Ctrl + C, then try again.
📁 Two files you'll meet:
  • package.json — your project's "ID card": name, scripts, and the list of packages it needs.
  • node_modules/ — the installed packages' actual code. Never edit by hand, never copy by USB — just run npm install to rebuild it.
