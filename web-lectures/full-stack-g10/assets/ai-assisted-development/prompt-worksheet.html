🤖 Prompt Worksheet

Turn vague questions into great AI prompts using the C-T-C-E structure. Type your answers, then compare with the model answer.

The C-T-C-E Structure

C
Context
Who are you, what are you building, what stack/tools?
T
Task
Exactly what do you want the AI to do?
C
Constraints
Rules: libraries, style, "must not use X", add comments, etc.
E
Examples
Sample data, or input → expected output.

Exercise 1: "Make my site look better"

❌ Vague: "make my site look better"

Rewrite it using C-T-C-E. (Hint: name your stack, say what page, give a goal.)

Show a model answer
✅ Good: "I'm a Grade 10 student building a barangay announcement page with HTML + Bulma. Right now it looks plain. Task: suggest a cleaner card-based layout for 3 announcement posts. Constraints: Bulma classes only, mobile-first, no JavaScript. Example: each card should show a title, date, and short text."

Exercise 2: "Fix the database"

❌ Vague: "fix the database"

This one needs the 4-part bug report (expected / actual / exact error / smallest code).

Show a model answer
✅ Good: "I have an Express + better-sqlite3 app. Expected: adding a resident saves to the `residents` table. Actual: server returns 500. Exact error: `SQLITE_ERROR: no such column: firts_name`. Smallest code: `INSERT INTO residents (first_name, age) VALUES (?, ?)` with `[name, age]`. My table was created with columns `id, first_name, age`. Why the error and how do I fix it?"

Exercise 3: "Add login"

❌ Vague: "add login"

Be specific about the method, constraints, and what you already have.

Show a model answer
✅ Good: "I have an Express + EJS + better-sqlite3 app with a `users` table (id, username, password_hash). Task: add login using server sessions (express-session) + bcrypt for password hashing — do NOT hand-roll crypto or use JWT. Constraints: beginner-friendly, comment each step, plain callbacks or async/await. I already have a login form posting to /login. Show the route handler and how to check the session on protected pages."

✅ Before You Trust the Answer — Verify

1. RUN it → does it actually work? 2. UNDERSTAND it → can you explain each line to a classmate? 3. Is it CURRENT? → does the API really exist? (check MDN / docs if unsure)

If you can't explain it, ask the AI to explain it — or read the docs. Don't ship code you don't understand.
